# 412_SQL_Implementation

Backend-first SQL implementation for the CSE 412 Phase 3 deliverable. The repo now contains:

- A canonical relational schema, seed dataset, and five assignment queries.
- MySQL-ready `ddl.sql`, `insert.sql`, `query.sql`, `drop.sql`, and `run.sql` files at the repo root.
- A local Python backend service that exposes the same data and queries in a frontend-ready shape.
- Automated tests that verify schema creation, referential integrity, cascade behavior, and query results.
- A generated `output.txt` transcript produced from the same SQL-compatible backend flow.

## Quick Start

Build the root SQL assets:

```powershell
python scripts/build_sql_assets.py
```

Generate the verification transcript:

```powershell
python scripts/generate_output.py
```

Run the automated tests:

```powershell
python -m unittest discover -s tests -v
```

## Project Layout

- `backend/catalog.py`: single source of truth for schema, seed rows, and assignment queries.
- `backend/service.py`: frontend-ready service methods for browsing tables and executing named queries.
- `backend/query_engine.py`: parameter compilation for SQLite today and MySQL integration later.
- `backend/sql_assets.py`: renders the assignment `.sql` files from the canonical catalog.
- `backend/output_generator.py`: produces `output.txt` from the same backend flow.
- `docs/assignment_coverage.md`: requirement-by-requirement implementation mapping.

## MySQL Note

The committed SQL files are written for MySQL-style execution and `run.sql` uses `SOURCE filename.sql;`, per the assignment note. A native MySQL server is not bundled inside this repository, so the checked-in `output.txt` is generated by the local verification runner from the same schema, seed data, and query catalog. When you are ready to submit, run the committed `run.sql` inside MySQL Workbench or the MySQL CLI to regenerate a native MySQL transcript.
